# 生前遗产寻宝长卷设计规格

日期：2026-07-21  
状态：等待产品负责人最终审阅  
适用地区：香港  
现有产品：`legacy-share/index.html`

## 1. 产品定位

将现有“亲属替已故先人分配虚拟遗产的公益游戏”重构为一段供临终者本人独立使用的生前遗产探索旅程。

产品的作用是帮助用户：

- 以轻松、尊重自主权的方式理解自己拥有的主要财产；
- 思考希望照顾的人及财产的大方向；
- 想象公益遗赠能够带来的社会影响；
- 在相关情境出现时，理解少量香港遗产法律常见误解；
- 形成可供本人、亲友及香港律师阅读的意愿摘要。

产品不是遗嘱制作工具，不生成可签署遗嘱，不提供针对个人情况的法律结论，也不代替香港执业律师。

## 2. 目标用户与体验原则

主要用户是临终者本人，并由本人独立操作。页面必须直接使用“你”“你的财产”“你希望照顾的人”等第一人称导向文案，不再将用户称作“先人”或让亲属推测其意愿。

体验必须遵循以下原则：

1. 用户可以只看故事，不回答任何问题，也能从头走到终点。
2. 滚动负责讲故事；点击负责自愿探索；只有用户主动选择安排某项财产时才要求输入。
3. 每个场景只传递一个重点，避免连续表格、法律术语及必填字段。
4. “不知道”“稍后再想”“暂时不填金额”都是正常完成状态。
5. 不以财富多少、公益比例或填写完整度评价用户。
6. 不诱导公益捐赠；选择不进行公益遗赠必须受到同等尊重。
7. 全流程使用大字体、高对比、大点击区域，并支持键盘及减少动画模式。

## 3. 产品范围

### 3.1 纳入范围

- 香港及海外物业；
- 现金、银行存款、股票、基金、债券及其他投资；
- 保险、强积金及退休权益；
- 公司、合伙或其他商业权益；
- 具有财产价值的数字资产；
- 珠宝、艺术品、收藏品及重要个人物品；
- 别人欠用户的款项；
- 按揭、贷款、信用卡及其他债务；
- 联名资产；
- 希望给予亲友、受养人或其他人士的财产；
- 公益及慈善遗赠；
- 尚未决定如何处理的剩余遗产。

### 3.2 明确排除

- 医疗决定及预设医疗指示；
- 持久授权书的制作；
- 殡葬、器官捐赠及纪念安排；
- 情感遗书；
- 正式遗嘱生成、签署或见证；
- 云端账号、服务器保存及个人结果分享链接。

## 4. 核心交互：纵向寻宝地图长卷

页面采用一张纵向延伸的成熟寻宝地图。用户用鼠标滚轮或手指向下移动，地图沿路线逐段展开，迷雾退去，区域及介绍依次出现。

地图不是儿童海盗游戏，而是带有东方航海长卷气质的“人生藏宝图”：深蓝夜色、旧纸纹理、金色路线、香港山海及城市轮廓。地图从夜色起步，在公益未来和终点逐渐过渡到晨光。

### 4.1 滚动行为

- 地图作为固定或半固定视觉层，叙事卡随滚动进入可视区域；
- 每个区域进入视口时，局部迷雾缓慢散开，金色路线延长；
- 动画只服务于方向感，不使用快速缩放、强烈视差、晃动或闪烁；
- 用户无需点击“下一页”；持续向下即可看到全部介绍；
- 提供“减少动画”和“直接查看完整地图”；
- 手机端改为纵向地图卡，但保留相同路线和内容顺序。

### 4.2 地图区域

1. **起点：我的一生**——建立本人自主规划的语境。
2. **家园岛**——物业、居住与持有方式。
3. **金库港**——现金、存款及投资。
4. **守护灯塔**——保险、强积金及退休权益。
5. **事业城**——公司、生意及商业权益。
6. **珍藏山谷**——重要物品、收藏及数字财产。
7. **远方群岛**——香港以外的资产。
8. **风浪海域**——按揭、贷款及其他债务。
9. **亲友村落**——希望照顾的人和主要方向。
10. **明日大陆**——公益遗赠与未来影响愿景。
11. **终点：我的方向**——完整地图及输出选择。

## 5. 三层参与方式

每个地图区域均提供三层参与深度：

### 5.1 只是看看

用户继续滚动，只阅读简短介绍，不记录资料。

### 5.2 这与我有关

用户点击一次，该地点亮起。系统只记录用户拥有或关心此类别，不要求金额或详细答案。

### 5.3 我想安排这里

用户主动点击后，地图旁展开一张小卡片。单次最多处理三个决定：

- 大概或精确价值；
- 希望该财产主要走向谁或用于什么；
- 现在填写详情或稍后补充。

输入卡关闭后，用户回到原滚动位置。任何未填写项目都不能阻挡继续探索。

## 6. 场景文案与故事节奏

### 6.1 序章

核心文案方向：

> 关于你的财产，不必等你离开以后再由别人猜测。这里会陪你看看自己一生累积下来的东西，想清楚希望照顾谁、留下些什么，以及能为世界带来什么。

页面简短说明：这是意愿整理工具，不是正式遗嘱；资料只在本机处理；用户可以随时暂停并下载草稿。

### 6.2 财产区域

每个区域只显示一段两至三句的精要介绍。例如家园岛：

> 家不只是一项财产，也可能是某个人未来安心生活的地方。它以什么方式持有，会影响它将来沿哪一条路线前进。

默认不展示完整专业问题。详情仅在用户点击“我想安排这里”后出现。

### 6.3 亲友村落

用户可以用称呼建立人物卡，例如伴侣、孩子、孙辈、依靠我的家人或其他重要的人。第一层只询问“你最希望为他／她留下什么”，选项包括生活保障、居住地方、特定物品、遗产的一部分或稍后再想。

姓名、替代受益人、未成年或特别管理需要等内容放入可选详情，不阻挡主路线。

### 6.4 明日大陆

公益方向以场景探索呈现，而不是偏好问卷。可探索场景包括教育、医疗公益、长者、基层家庭、动物、自然、文化及社区。这里的“医疗公益”只指遗赠希望帮助的社会方向，不收集或处理用户本人的医疗决定。

点击场景先显示一段未来愿景，再询问是否把它加入个人地图。用户可选择固定金额、遗产比例、只记录愿望或不进行公益遗赠。

所有未来故事必须标记为愿景示例，不得承诺真实受益人数、永久冠名、投资收益或持续年限。

## 7. 香港法律教育：地图路标

法律内容采用情境触发的路标、灯塔注释或航海提示。每次只显示一个重点和最多两三句说明。

示例：

> 小路标：联名财产未必沿着遗嘱所写的路线前进。

用户可选择“知道了”或“为什么？”。只有点击“为什么？”才展开解释及香港官方来源链接。

提示必须基于用户已点亮的区域：

- 联名物业或账户：提示持有方式可能影响财产传递；
- 保险、强积金：提示需要向有关机构或香港律师确认处理方式；
- 海外资产：提示可能需要跨境专业意见；
- 债务：提示遗产通常须先处理债务及相关开支；
- 未成年或依赖照顾者：提示可能需要适当管理安排；
- 公益遗赠：提示机构身份、用途及替代安排需要写清楚。

法律资料发布前必须用最新香港官方来源核实，优先采用香港电子法例、司法机构、政府部门、强积金管理局及相关监管机构资料，并显示最后核实日期。不能把律师楼营销文章作为唯一依据，也不能根据用户答案给出个人法律结论。

## 8. 地图状态与完成概念

不显示完成百分比、欠缺题目数量或红色错误清单。地图地点只显示：

- 已探索；
- 已记下心愿；
- 稍后确认。

即使用户没有输入任何金额，也可以抵达终点。系统以温和语言说明哪些方向已经表达、哪些内容可在与律师见面前补充。

## 9. 终点与输出

终点显示三条可能路线：走向希望照顾的人、走向公益未来、仍等待决定的部分。路线缓慢亮起，不使用彩带、奖章、分数或财富称号。

终点核心文案：

> 你留下的不只是财产，而是你亲自选择的方向。

### 9.1 我的心愿地图 PDF

给本人及亲友阅读的简洁版本，包括：

- 我最重视的事；
- 我希望照顾的人；
- 主要财产方向；
- 公益愿望；
- 尚未决定的部分；
- 非遗嘱及无法律效力声明。

默认不显示精确金额，用户可主动选择包含金额。

### 9.2 给香港律师看的 PDF

包含用户主动填写的详细资料：资产类别及金额、债务、分配方向、公益意愿、地图触发的法律确认事项，以及建议进一步询问的问题。每页明确标记为“意愿整理资料，不是遗嘱，也没有法律效力”。

### 9.3 不提供分享链接

个人心愿、资产及金额不生成网络链接、不编码进网址、不上传服务器。网站可以继续公开访问，但用户结果只能由本人以本机文件方式保存或分享。

## 10. 草稿保存与隐私

所有资料只在浏览器本机处理，不使用第三方分析、广告追踪、外部 AI 或服务器储存。

提供：

- **轻量草稿**：保存探索位置、已点亮区域及主要心愿，不包含精确金额；
- **完整加密草稿**：包含精确金额，使用用户设置的密码在浏览器本机加密后下载。

重新打开网站后，用户通过“继续我的旅程”导入草稿。导入时必须验证文件版本、格式和数据结构；损坏、密码错误或不兼容文件应显示清楚且不破坏当前页面状态。

页面必须提供“清除所有资料”。离开页面时，仅在存在未保存的用户输入时提示资料可能消失。

## 11. 视觉素材系统

主要视觉采用统一的温暖电影感 AI 插画：半写实、成熟、含蓄，避免儿童卡通及海盗符号。人物以背影、剪影或生活场景为主，不使用病床、墓地、哭泣老人或制造恐惧的画面。

网上素材仅用于纸张、云雾、星空等辅助纹理，并且必须是许可清楚的公共领域或 CC0 资源。项目内需记录素材来源及许可。不得把“免费下载”等同于无版权限制。

每个区域只配置一张主要场景视觉，避免图片过多造成认知负担。图片应有替代文字，纯装饰纹理应从辅助技术中隐藏。

## 12. 技术结构

产品继续保持静态部署及 GitHub Pages 兼容，但不再把所有状态、法律文案、场景数据和渲染逻辑混在单一全局对象中。

建议逻辑边界：

- `mapScenes`：区域、叙事、视觉及滚动触发配置；
- `journeyState`：已探索、已记录和稍后确认状态；
- `estateData`：用户主动填写的资产、债务及方向；
- `legalSignals`：由已选择情况触发的香港法律路标；
- `draftCodec`：轻量草稿、加密草稿及导入验证；
- `reportRenderer`：简版及律师版 PDF／打印输出；
- `accessibilityController`：减少动画、键盘焦点及状态播报。

金额统一使用港币并进行基本验证；不允许负数、非数值或静默超额分配。用户输入错误时保留原内容并提供简短修正提示。

## 13. 错误与降级处理

- JavaScript、图片或动画失败时，所有介绍及按钮仍能以普通纵向内容使用；
- 不支持滚动动画的设备显示完整地图卡片；
- PDF生成失败时提供浏览器打印版；
- 图片加载失败显示替代文字或纯色场景，不阻挡主流程；
- 草稿导入失败不覆盖现有资料；
- 浏览器不支持加密能力时，不提供“完整加密草稿”，并明确说明原因，不降级成未加密敏感草稿。

## 14. 验收与测试

必须验证：

1. 不输入任何资料也能滚动到终点；
2. 每个区域的“只是看看”“这与我有关”“我想安排这里”均可正常使用；
3. 点击详情后关闭可回到原滚动位置；
4. 地图迷雾、路线及动画在桌面和手机正常降级；
5. 减少动画模式不影响内容理解；
6. 键盘、屏幕阅读器及大字体模式可操作；
7. 法律路标只在相关情境出现；
8. 选择不进行公益遗赠可正常完成；
9. 轻量草稿导出和导入一致；
10. 完整加密草稿可正确加密、解密，并妥善处理错误密码；
11. 简版 PDF 默认不含精确金额；
12. 律师版 PDF 包含用户主动填写的细节及免责声明；
13. 页面及网址不泄露个人资料；
14. 浏览器网络检查确认不会传送用户输入；
15. GitHub Pages 上线后完成从起点到终点的真实浏览器测试。

## 15. 已确认的产品决定

- 用户：香港临终者本人独立使用；
- 定位：意愿整理与法律误解教育，不生成正式遗嘱；
- 范围：整个遗产与公益遗赠，不处理医疗及殡葬事项；
- 体验：故事长卷，不是问卷；
- 地图：寻宝地图风格，滚动逐段揭开迷雾；
- 互动：滚动看故事，点击才记录，主动安排才填写；
- 金额：允许精确金额，但非必填；
- 法律内容：边探索边显示短路标；
- 素材：统一 AI 插画为主，许可明确的辅助纹理为辅；
- 草稿：支持轻量草稿及完整加密草稿导入／导出；
- 输出：简版 PDF 及律师版 PDF；
- 分享：不提供个人结果分享链接；
- 部署：继续使用静态网站及 GitHub Pages。
